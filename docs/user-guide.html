🖥️ IIS3DWB GUI 사용 설명서

설정 툴로 디바이스 WiFi를 설정하는 방법 — 따라하기

IIS3DWB 설정 툴 · macOS
이 문서는 펌웨어가 이미 올라간 디바이스의 WiFi 설정용입니다. 펌웨어를 처음 굽는 셋업은 셋업 설명서를 보세요.
📡 반드시 2.4GHz WiFi를 사용하세요
이 디바이스(ESP32-S3)는 2.4GHz WiFi만 지원합니다. 5GHz WiFi는 디바이스가 아예 인식하지 못해 연결되지 않습니다.
WiFi 이름 예시대역연결
example2.4G2.4GHz✅ 연결됨
shinho5G5GHz❌ 연결 안 됨
💡 공유기가 듀얼밴드(2.4G+5G)면, 이름 끝에 2.4G/2G가 붙거나 5G 표시가 없는 쪽을 고르세요. 5GHz WiFi를 입력하면 툴이 "WiFi를 찾을 수 없습니다"로 안내합니다.
목차 1. 툴 실행하기 2. [1단계] 연결 — ★포트 선택값★ 3. [2단계] WiFi 입력 4. [3단계] 완료 확인 5. 한눈에 보는 "내가 선택/입력할 값" 6. 문제 해결

1 툴 실행하기

중요: 툴을 켜기 전에, 같은 디바이스를 쓰는 다른 프로그램 (idf.py monitor, 시리얼 터미널 등)을 반드시 종료하세요. USB 포트는 한 번에 하나의 프로그램만 쓸 수 있습니다.

개발 모드로 실행 (현재)

cd /Users/kimkookjin/Projects/ESP-IDF/IIS3DWB/config-tool
npm run tauri dev

잠시 후 "IIS3DWB 진동센서 WiFi 설정" 창이 뜹니다 (560×720).

① 디바이스를 USB로 PC에 연결한 상태에서 ② 위 명령을 실행하면 됩니다.

2 [1단계] 연결 — ★ 포트 선택값 ★

🔌
디바이스를 USB로 연결하세요
/dev/cu.usbmodem1433301▼
👆 여기를 클릭해 목록에서 선택
디바이스 연결
📌 "USB 포트" 목록에서 무엇을 고르나요?

usbmodem이 포함된 항목을 고르세요. 이것이 ESP32-S3 디바이스입니다.

목록에 보이는 값선택?설명
/dev/cu.usbmodem1433301✅ 이것ESP32-S3 디바이스 (숫자는 다를 수 있음)
/dev/cu.usbserial-XXXX✅ 가능일부 보드는 usbserial로 표시됨
/dev/cu.Bluetooth-...❌ 아님블루투스 — 디바이스 아님 (목록에서 자동 제외됨)

💡 보통 디바이스가 1개면 목록에 하나만 뜹니다. 그걸 그대로 두면 됩니다. 목록이 비어 있으면 ⟳ (새로고침) 버튼을 누르세요.

"디바이스 연결" 버튼을 누르면

툴이 그 포트로 연결해 디바이스 정보를 읽습니다. 성공하면 녹색 박스로 표시됩니다:

✅ 디바이스 감지됨
칩: ESP32-S3
MAC: 30:ED:A0:1A:89:30
기능: WiFi, BLE
WiFi 설정 ▶

→ "WiFi 설정 ▶" 버튼을 눌러 2단계로 넘어갑니다.

감지 실패(빨간 박스)면: 포트를 잘못 골랐거나, 다른 프로그램이 포트를 쓰는 중이거나, 케이블 문제입니다. 다른 프로그램을 닫고 다시 시도하세요.

WiFi 비밀번호       👁
👁 눈 아이콘으로 입력값 보기/숨기기

WiFi 이름 (SSID)디바이스를 연결할 WiFi 이름을 직접 타이핑. 대소문자 정확히! (예: example2.4G)
비밀번호그 WiFi의 비밀번호. 오픈 WiFi면 비워둠.
2.4GHz WiFi만 됩니다. ESP32-S3는 5GHz를 지원하지 않습니다. 공유기가 듀얼밴드면 2.4GHz 대역의 SSID를 입력하세요.

"저장 & 연결 확인" 버튼을 누르면, 툴이: ① WiFi 정보를 디바이스 메모리(NVS)에 저장 → ② 디바이스 재부팅 → ③ 실제로 그 WiFi에 연결되는지 확인 (최대 20초)합니다.

연결은 보통 바로 안 붙습니다. 디바이스는 1~5초 간격으로 자동 재시도하며 연결합니다. 그래서 툴은 최대 60초까지 기다리며 결과를 판정합니다. 잠시 기다려 주세요.
📌 완료 화면의 버튼 — 결과에 따라 다릅니다
결과강조 버튼보조 버튼

💡 성공하면 "종료"를 눌러 끝내면 됩니다. 실패하면 "다시 설정"이 강조되니, 값을 고쳐 다시 시도하세요. 여러 디바이스를 연속 설정할 때는 "다른 디바이스 설정"을 누르세요.

결과는 4가지로 명확히 안내됩니다:

화면의미할 일
✅ 설정 완료 & WiFi 연결됨!
+ IP 주소		 성공. 연결 확인됨. USB 분리 OK. 종료 클릭.
❌ WiFi를 찾을 수 없습니다 주변에 그 WiFi가 없음.
5GHz이거나 이름 오타일 가능성 큼.		 2.4GHz WiFi인지, 이름 대소문자가 맞는지 확인 후 다시 설정
🔑 비밀번호 오류로 보입니다 WiFi는 찾았으나 인증 실패. 👁 아이콘으로 비밀번호 확인 후 다시 설정
⏳ 저장 완료 — 연결 시도 중 저장됨. 신호 약함/공유기 바쁨 등으로 60초 내 연결 확인 못함.
디바이스는 계속 자동 재시도 중.		 잠시 후 공유기에서 연결 확인, 또는 신호 강한 곳으로 이동
⏳·🔑·❌ 모두 WiFi 정보는 이미 디바이스에 저장되어 있습니다. 값이 맞고 신호만 약했던 경우라면, 시간이 지나면 자동으로 연결됩니다. 값 자체가 틀렸으면 다시 설정으로 올바른 값을 넣으세요.
⚠️ 5GHz 주의: 이름에 5G가 붙은 WiFi(예: shinho5G)는 연결되지 않습니다. ESP32-S3는 2.4GHz만 지원합니다. example2.4G 같은 2.4GHz 대역 이름을 사용하세요.

▸ 부팅 로그 보기를 펼치면 디바이스의 연결 과정·재시도·실패 사유를 볼 수 있습니다 (진단용).

성공 시, 이후로는 디바이스에 전원만 들어오면 저장된 WiFi로 자동 연결됩니다. WiFi를 바꿀 때만 이 툴로 다시 설정하면 됩니다.

5 한눈에 보는 "내가 선택/입력할 값"

단계요소종류내가 할 것
1USB 포트목록(list box)usbmodem 포함 항목 선택 (보통 자동으로 1개)
1디바이스 연결버튼클릭 → 녹색 박스 뜨면 OK
1WiFi 설정 ▶버튼클릭 (2단계로 이동)
2WiFi 이름 (SSID)텍스트 입력WiFi 이름 직접 타이핑 (2.4GHz)
2비밀번호텍스트 입력WiFi 비밀번호 입력
2저장 & 연결 확인버튼클릭 → 최대 60초 대기 (자동 재시도하며 연결)
3종료 / 다시 설정버튼성공 → 종료 / 실패 → 다시 설정 (결과에 따라 강조됨)

6 문제 해결

증상원인 / 해결
포트 목록이 비어 있음USB 연결 확인 → ⟳ 새로고침 버튼 클릭. 케이블이 데이터용인지 확인.
"연결 실패" 빨간 박스 (1단계)다른 프로그램(monitor/터미널)이 포트 점유 중 → 종료 후 재시도. 포트 잘못 선택했는지 확인.
❌ "WiFi를 찾을 수 없습니다"5GHz WiFi이거나 이름 오타 → 2.4GHz 이름·대소문자 확인 후 다시 설정
🔑 "비밀번호 오류로 보입니다"비밀번호 확인(👁) 후 다시 설정
⏳ "저장 완료 — 연결 시도 중"신호 약함/공유기 바쁨 → 디바이스가 자동 재시도 중. 신호 강한 곳으로 이동하거나 잠시 대기
창이 안 뜸npm run tauri dev 가 빌드 중일 수 있음. 잠시 대기.
여러 포트가 보임usbmodem/usbserial 포함 항목 선택. 모르면 하나씩 "디바이스 연결" 시도.
