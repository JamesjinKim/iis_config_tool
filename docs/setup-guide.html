ESP32-S3 · ESP-IDF v5.4.3 · 1회만 수행
이 문서는 최초 1회 셋업(펌웨어를 디바이스에 굽기)용입니다. 펌웨어가 이미 올라간 디바이스의 WiFi 설정은 GUI 사용 설명서를 보세요.
펌웨어 굽기 방법은 2가지입니다. 본인 상황에 맞게 고르세요.
방법대상ESP-IDF 필요?
방법 A — GUI 툴로 설치 (권장)일반 사용자, 빠른 설치❌ 불필요
방법 B — 터미널(idf.py)개발자, 코드 수정·재빌드✅ 필요
목차 방법 A. GUI 툴로 펌웨어 설치 (권장) [방법 B] 1. 준비물 [방법 B] 2. 개발 환경 활성화 [방법 B] 3. 디바이스 연결 & 포트 확인 [방법 B] 4. 펌웨어 빌드 & 플래시 5. 정상 동작 확인 (공통) 6. 문제 해결 7. 회사용 — 제품별 GUI 빌드 / 버전 관리

A GUI 툴로 펌웨어 설치 (권장)

설정 툴에 펌웨어가 내장되어 있어, ESP-IDF 없이 버튼 클릭만으로 설치됩니다.

1
디바이스를 USB로 연결하고 설정 툴을 실행합니다. 
cd /Users/kimkookjin/Projects/ESP-IDF/IIS3DWB/config-tool
npm run tauri dev

(배포된 .app/.dmg가 있으면 그냥 더블클릭 실행)

2
1단계에서 USB 포트 선택 → "디바이스 연결" 클릭. 디바이스가 감지되면 "⚙ 펌웨어 설치 / 업데이트" 버튼이 나타납니다.
3
"⚙ 펌웨어 설치 / 업데이트"를 누르고 확인하면, 펌웨어가 자동으로 구워집니다 (1~2분).
4
"✅ 펌웨어 설치 완료"가 뜨면, 그대로 "WiFi 설정 ▶"으로 진행하면 됩니다. (이후는 GUI 사용 설명서 참고)
방법 A로 설치하면 펌웨어 설치 + WiFi 설정을 툴 하나에서 모두 끝낼 수 있습니다.
⚠️ 펌웨어 설치 중에는 USB를 분리하지 마세요. 다른 프로그램(시리얼 모니터 등)이 포트를 쓰고 있으면 먼저 종료하세요.

아래부터는 방법 B (터미널/개발자용)입니다. 방법 A로 설치했다면 5. 정상 동작 확인으로 건너뛰세요.

1 준비물

PCmacOS (이 문서 기준)
ESP-IDFv5.4.3 설치됨 (~/esp/v5.4.3/esp-idf)
프로젝트/Users/kimkookjin/Projects/ESP-IDF/IIS3DWB
셋업(펌웨어 플래시)은 ESP-IDF가 필요합니다. 일반 사용자가 WiFi만 바꿀 때는 GUI 툴만 있으면 되고 ESP-IDF는 필요 없습니다.

2 개발 환경 활성화

새 터미널을 열 때마다 한 번 실행합니다.

get_idf
cd /Users/kimkookjin/Projects/ESP-IDF/IIS3DWB

get_idf는 source ~/esp/v5.4.3/esp-idf/export.sh의 단축 별명입니다. 실행 후 idf.py 명령을 쓸 수 있습니다.

3 디바이스 연결 & 포트 확인

1
ESP32-S3 보드를 USB 케이블로 PC에 연결합니다.
2
포트가 인식됐는지 확인: 
ls /dev/cu.usbmodem*

→ /dev/cu.usbmodem1433301 같은 포트가 나오면 정상. 숫자는 환경마다 다를 수 있습니다.

포트가 안 보이면: 케이블 교체(데이터 전송용인지 확인), USB 허브 대신 PC 직결, 다른 프로그램이 포트를 쓰고 있는지 확인.

4 펌웨어 빌드 & 플래시

1
빌드 (첫 빌드는 수 분, 재빌드는 약 30초) 
idf.py build
2
플래시 (포트는 본인 환경에 맞게) 
idf.py -p /dev/cu.usbmodem1433301 flash
Hash of data verified. 와 Hard resetting... 가 보이면 플래시 성공. 디바이스가 자동 재부팅됩니다.

5 정상 동작 확인

시리얼 모니터로 부팅 로그를 확인합니다.

idf.py -p /dev/cu.usbmodem1433301 monitor

아래와 비슷한 로그가 보이면 정상입니다. (종료: Ctrl + ])

I CONFIG_MGR: 설정 관리자 초기화 완료
I SERIAL_PROTO: 시리얼 프로토콜 태스크 시작
  ⚙ WiFi 설정이 없습니다. 설정 대기 모드로 진입합니다.   ← WiFi 미설정 상태 (정상)
I IIS3DWB: Device ID verified: 0x7B                      ← 센서 인식 성공
  ★ IIS3DWB Sensor Ready!
I IIS3DWB: [1] Accel: X= ... Mag= ...                    ← 센서 측정 동작
"WiFi 설정이 없습니다 → 설정 대기 모드"는 정상입니다. 이제 GUI 툴로 WiFi를 설정하면 됩니다.

6 문제 해결

증상해결
idf.py: command not foundget_idf를 먼저 실행 (환경 미활성화)
포트를 못 찾음케이블/USB 직결 확인, ls /dev/cu.usbmodem* 재확인
플래시 중 연결 실패다른 프로그램(monitor/GUI툴)이 포트 점유 중인지 확인 후 종료
센서 ID 실패 (0x7B 아님)SPI 배선(MOSI/MISO/SCLK/CS) 점검
부팅 반복 (ESP-ROM 계속 출력)idf.py -p PORT erase-flash 후 재플래시

7 회사용 — 제품별 GUI 빌드 / 버전 관리

이 GUI 툴은 펌웨어가 내장된 상태로 배포됩니다. 회사는 제품 버전에 맞는 펌웨어를 GUI에 넣어 빌드한 뒤, 사용자가 자기 제품에 맞는 GUI를 받아 셋업합니다.

[회사] 펌웨어 빌드 → GUI에 번들 → 제품별 GUI(.dmg) 빌드 → 배포 [사용자] 제품 구매 → 자기 제품에 맞는 GUI 다운로드 → 셋업

① 펌웨어를 GUI에 번들

펌웨어 빌드 후, 산출 .bin 3개를 GUI 프로젝트의 firmware/ 폴더로 복사합니다.

cd /Users/kimkookjin/Projects/ESP-IDF/IIS3DWB
idf.py build      # 펌웨어 빌드

# 산출물을 GUI 번들 폴더로 복사
cp build/bootloader/bootloader.bin       config-tool/src-tauri/firmware/bootloader.bin
cp build/partition_table/partition-table.bin config-tool/src-tauri/firmware/partition-table.bin
cp build/tsl2591_sensor.bin              config-tool/src-tauri/firmware/app.bin

② 버전 정보 기입 (★중요)

GUI에 표시될 제품/버전을 config-tool/src-tauri/src/lib.rs 상단 상수 블록에서 수정합니다. 사용자는 이 정보로 "내 제품에 맞는 GUI인지" 확인합니다.

const PRODUCT_NAME: &str = "IIS3DWB 진동센서";       // 제품명
const PRODUCT_MODEL: &str = "IIS3DWB-VIB-SENSOR";   // 모델 코드
const FIRMWARE_VERSION: &str = "1.0.0";            // ← 제품 릴리스 버전
const ESP_IDF_VERSION: &str = "v5.4.3";            // 빌드에 쓴 IDF 버전
const TARGET_CHIP: &str = "ESP32-S3";

이 값은 GUI 하단 푸터와 "펌웨어 설치 완료" 메시지에 표시됩니다.

③ 배포용 GUI 빌드 (.dmg)

cd config-tool
npm run tauri build      # → src-tauri/target/release/bundle/ 에 .app / .dmg 생성
완성된 .dmg는 ESP-IDF 없이 동작합니다. 사용자에게 배포하면 더블클릭 실행 → 펌웨어 설치 + WiFi 설정까지 모두 가능합니다.
⚠️ 버전 일관성: 펌웨어를 새로 빌드했으면 ①(firmware 복사)와 ②(버전 기입)를 반드시 함께 갱신하세요. 그래야 사용자가 보는 버전과 실제 설치 펌웨어가 일치합니다.
