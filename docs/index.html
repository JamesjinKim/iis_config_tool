1 한 줄 요약

핵심 성과: PC에서 WiFi 값이 담긴 NVS 바이너리를 만들어 Flash에 직접 주입하면, 디바이스가 부팅 시 그 값을 읽어 WiFi에 자동 연결한다. 실기기에서 IP 192.168.0.225 획득까지 검증 완료. 제품의 핵심 가치 — "한 번 설정하면 영구 저장되어 자동 연결" — 이 실증되었다.
이 방식은 USB 실시간 통신(RX) 없이도 동작하므로, 개발 검증은 물론 공장 양산 시 일괄 WiFi 주입에도 그대로 활용할 수 있다.

3 시스템 구조

[PC] WiFi 값 입력 │ ├─ nvs_partition_gen.py → nvs_wifi.bin (NVS 바이너리) │ └─ esptool → Flash 0x9000 (NVS 파티션)에 주입 │ ▼ [ESP32-S3 + IIS3DWB] 부팅 → NVS에서 WiFi 읽기 → 자동 연결 → 센서 측정 결과는 USB 로그(TX)로 확인

4 저장 위치 — NVS(Flash)

WiFi 설정은 센서(IIS3DWB) 내부가 아니라, ESP32-S3 모듈의 SPI Flash 칩 안 NVS 파티션에 저장된다.

항목값
파티션nvs (data/nvs)
오프셋0x9000
크기24K (0x6000)
네임스페이스devcfg
키wifi_ssid (string), wifi_pass (string)
NVS는 펌웨어(앱 코드)와 물리적으로 분리된 파티션이라, 펌웨어를 재플래시해도 설정이 남는다. 전원이 꺼져도 유지(비휘발성)되어 재부팅 후 자동 연결이 가능하다. (EEPROM이라는 별도 칩은 이 보드에 없으며, 그 역할을 Flash 내 NVS가 한다.)

5 성공 방법 — NVS 직접 주입

사용자가 입력한 WiFi 값을 NVS 형식의 바이너리로 변환한 뒤, esptool로 NVS 파티션 위치에 굽는다. 디바이스는 이미 그 위치에서 설정을 읽도록 만들어져 있으므로, 추가 통신 없이 자동 연결된다.

CSV (key,type,encoding,value) │ nvs_partition_gen.py generate ▼ nvs_wifi.bin (24K NVS 이미지) │ esptool write_flash 0x9000 ▼ 디바이스 Flash NVS 파티션 → 부팅 시 읽힘 → 자동 연결

NVS CSV 형식 (검증된 형식)

key,type,encoding,value
devcfg,namespace,,
wifi_ssid,data,string,example2.4G
wifi_pass,data,string,example1234
네임스페이스 devcfg와 키 이름(wifi_ssid, wifi_pass)은 펌웨어 config_manager.c의 정의와 정확히 일치해야 한다. 문자열은 반드시 type=data, encoding=string으로 지정 (펌웨어가 nvs_get_str로 읽기 때문).

6 따라하기 — 단계별

파일 위치: config-tool/ · 디바이스 포트(macOS): /dev/cu.usbmodem1433301

1환경 활성화

get_idf
cd /Users/kimkookjin/Projects/ESP-IDF/IIS3DWB/config-tool

2WiFi 값을 CSV에 작성

nvs_wifi.csv 파일의 SSID/비밀번호를 원하는 값으로 수정.

3NVS 바이너리 생성

python $IDF_PATH/components/nvs_flash/nvs_partition_generator/nvs_partition_gen.py \
       generate nvs_wifi.csv nvs_wifi.bin 0x6000

→ nvs_wifi.bin (24K) 생성됨

4NVS 파티션에 주입 (펌웨어는 유지)

esptool.py --chip esp32s3 -p /dev/cu.usbmodem1433301 -b 460800 \
           write_flash 0x9000 nvs_wifi.bin

→ 0x9000(NVS 오프셋)에만 쓰기. 펌웨어/부트로더는 건드리지 않음.

5결과 확인 (자동 연결)

idf.py -p /dev/cu.usbmodem1433301 monitor   # 또는 부팅 로그 캡처

로그에 ★ WiFi Connected Successfully! 와 IP 주소가 뜨면 성공. (모니터 종료: Ctrl+])

7 디바이스 부팅 동작

부팅 → NVS에서 WiFi 설정 읽기 (config_manager_load_wifi) ├─ 설정 있음 → 자동 WiFi 연결 → [정상 동작] 센서 측정 └─ 설정 없음 → [설정 대기 모드] (자동 연결 시도 안 함 → 크래시 없음)

8 펌웨어 구성 요소

ESP-IDF 빌드에 묶여 프로젝트 루트 components/에 위치 (설정 툴 폴더와 별개).

컴포넌트역할
components/config_manager/NVS(Flash) WiFi 저장·로드·초기화. 네임스페이스 devcfg
components/wifi_manager/WiFi 연결. 미설정 시 자동 연결 안 함(auto_connect 플래그)
main/main.c부팅 시 NVS 로드 → 있으면 자동연결, 없으면 대기
components/serial_protocol/USB JSON 명령 처리 (RX 통신용 — 현재는 NVS 주입 방식으로 우회)

항목내용
